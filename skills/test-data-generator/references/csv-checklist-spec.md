# 测试数据清单 CSV 规范

这份 CSV 面向测试人员直接阅读和使用，不追求数据库导入语义，重点是帮助测试人员准备数据并验证测试用例。

## 文件定位

- Markdown：负责完整的数据清单、基础记录池、待确认项和上下文说明
- CSV：负责测试执行时的快速查阅和逐条核对

建议文件名：

- `<需求名称>_测试数据清单.csv`

## 固定列

- `测试用例编号`
- `测试功能点`
- `数据目标`
- `涉及表`
- `关键字段`
- `建议数据值`
- `样例来源`
- `数据准备方式`
- `验证方式`
- `备注/待确认项`

## 列说明

### 测试用例编号

- 必须能对应到确认版 Markdown 清单中的 `TC-xxx`
- 不允许在 CSV 中出现 Markdown 没有的测试用例编号

### 测试功能点

- 用测试人员熟悉的话描述即可
- 不必完全复制需求标题，但应与测试用例意图一致

### 数据目标

- 直接说明这条数据是为了验证什么
- 建议写成“构造可按账号筛选的订单”“构造状态为 DCD 的订单”这类短句

### 涉及表

- 用逗号分隔表名
- 只写当前验证真正相关的表

### 关键字段

- 用逗号分隔字段名
- 只保留测试人员在准备或核对时必须关注的字段

### 建议数据值

- 建议使用 `字段=值` 的简洁形式
- 一个单元格内允许写多个 `字段=值`
- 不要求写全量字段，只写关键验证值

### 样例来源

- 写清来自哪个 `基础记录池` 或哪个 `table_samples/*.csv`
- 示例：`A1: ORDER_011`、`table_samples/orders.csv: ORDER_011`

### 数据准备方式

建议统一使用以下少量取值：

- `直接复用`
- `补造`
- `修改现有样例`
- `缺失态保留`

### 验证方式

- 必须直接说明测试人员如何用这条数据验证用例
- 应写成可执行句子，例如：`列表按账号查询应命中该记录`

### 备注/待确认项

- 已确认的说明可写简短备注
- 未确认规则不应进入正式 CSV，应只保留在 Markdown 的 `待确认项`

## 编码与格式

- 使用 `UTF-8 with BOM`
- 使用中文列名
- 保持一行一条记录，避免跨行复杂结构

## 与 Markdown 的关系

- CSV 是确认版 Markdown 的派生清单，不是独立真相源
- 如果 Markdown 与 CSV 冲突，以 Markdown 确认版为准
- 修改 CSV 前，优先确认是否应先回改 Markdown 清单稿
- 当前校验脚本会从 Markdown 中抽取 `TC-xxx` 顺序；如需派生 CSV，先保证 Markdown 清单稿结构完整
