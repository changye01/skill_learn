# 覆盖检查清单

生成场景地图后，必须逐项自查：

- 显式需求点是否都已映射到场景分组
- 是否包含正常流程
- 是否包含边界条件
- 是否包含异常场景
- 是否包含权限控制
- 是否包含状态流转
- 是否包含日志校验
- 是否包含数据一致性校验
- 是否包含数据持久化校验
- 是否包含历史数据处理校验
- 是否至少有一组综合场景或端到端场景
- 用户补充确认过的明确规则是否已回填到 `本次确认规则`
- 是否标记了信息不足待确认项
- 是否标记了超范围项
- 若存在明确权限角色列表，是否每个允许角色都有正向覆盖，且至少有 1 条无权限反向覆盖
- 若存在少量关键枚举值，是否按值展开到足够颗粒度，而不是粗暴合并
- 是否存在明显重复用例
- 是否存在颗粒度过粗、一步覆盖太多需求点的场景
- 场景地图正式文件是否只保留确认版内容，是否不存在重复标题或重复章节
