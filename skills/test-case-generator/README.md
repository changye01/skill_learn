# Test Case Generator Skill

一个基于需求文档自动生成测试场景和结构化测试用例的 Skill。

## 目标

把“需求文档 -> 可执行测试用例”的过程拆成两个清晰阶段：

1. 先生成场景地图，确保覆盖面正确。
2. 再展开结构化测试用例，确保输出可执行。

默认目标是`覆盖全面`，但不以数量最大化为目标。

## 适用范围

适用于以下场景：

- 根据需求文档生成功能测试用例
- 结合技术文档、接口文档、原型、已有用例补齐覆盖
- 先做场景评审，再输出结构化测试用例

默认不包含以下内容，除非用户明确要求：

- 性能测试
- 兼容性测试
- 安全测试
- 测试数据生成
- 操作手册生成

## 设计原则

- 先场景，后用例
- 覆盖全面，但不以数量最大化为目标
- 同时考虑显式需求和隐含影响点
- 默认补齐权限、日志、数据一致性、数据持久化、综合流程
- 超范围项只提示，不默认混入功能测试主体
- 禁止把大量仅有枚举值差异的重复内容机械拆成多条用例

## 两阶段生成流程

### 第一阶段：场景地图

先输出 Markdown 场景地图，重点回答“测什么”。

内容包括：

- 功能模块
- 需求点列表
- 场景分组
- 补充维度
- 信息不足待确认项
- 超范围项

### 第二阶段：结构化测试用例

在用户确认场景地图后，再展开结构化测试用例，重点回答“怎么测”。

内容包括：

- 需求名称
- 用例编号
- 模块
- 场景组
- 用例标题
- 关联需求点
- 前置条件
- 测试步骤
- 预期结果
- 优先级
- 备注

## 输出物说明

- `assets/scene-template.md`
  第一阶段场景地图模板
- `assets/test-case-template.md`
  第二阶段结构化测试用例模板

## 规则来源

- 测试设计规则：`references/test-design-rules.md`
- 覆盖检查清单：`references/coverage-checklist.md`
- 输出校验脚本：`scripts/validate_cases.py`

## 使用方式

建议输入顺序：

1. 需求文档
2. 技术文档、接口文档、原型、数据库说明等补充材料
3. 生成目标和范围约束

建议执行顺序：

1. 先生成场景地图
2. 确认场景地图
3. 再生成结构化测试用例
4. 运行校验脚本检查完整性

## 验证方式

对结构化测试用例文档运行：

```bash
python scripts/validate_cases.py <测试用例文件路径>
```

脚本会检查：

- 是否包含关键字段
- 是否包含综合场景或端到端场景

第一阶段场景地图默认依赖模板约束和人工确认，不纳入第一版脚本校验范围。
